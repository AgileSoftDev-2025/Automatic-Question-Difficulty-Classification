Help Center & FAQ

Have questions? We have answers. Find everything you need to know about Bloomers right here.

Bloomers is a web application designed to help educators and content creators. Its primary function is to automatically analyze and classify questions based on Bloom's Taxonomy (C1-C6), helping to ensure cognitive diversity in evaluation materials.

Based on `views.py` (`ALLOWED_EXTENSIONS`), we currently support the following file formats:

  • .txt (Plain Text)
  • .pdf (Portable Document Format)
  • .csv (Comma-Separated Values)
  • .docx (Microsoft Word Document)

The maximum file size you can upload is 10MB. This limit is set in `views.py` (`MAX_FILE_SIZE`).

An upload can fail for several common reasons, which are checked by our `validate_file` function:

  • Unsupported Format: You uploaded a file type other than .txt, .pdf, .csv, or .docx.
  • Size Exceeded: Your file is larger than the 10MB maximum limit.
  • Empty File: The file you uploaded has no content (0 bytes in size).
  • No File Selected: You clicked "Classify" without selecting a file first.

You don't *need* an account to try a classification. However, there is a big difference (as seen in `handle_file_upload`):

  • Guest (Anonymous) Users: Your file is processed, but no history is saved. The result is temporary and stored in the server cache (`handle_anonymous_upload`) for up to 1 hour.
  • Registered Users: Your file is saved permanently to your account (`handle_authenticated_upload`). You get a complete classification history, can track statuses, and re-download your original files at any time.

When you upload a file as a registered user, its initial status is set to 'pending'. This means your file has been successfully received and is waiting in the queue to be processed by our classification system. This process is usually fast but may take a few moments depending on server load.

Please refresh the page after a short time to see the updated status (e.g., 'Completed' or 'Failed').

  • Download: (The `download_file` function) This button lets you download the original file you previously uploaded. It is *not* a results report.
  • Delete: (The `delete_history` function) This button will permanently delete the history record from the database AND its associated physical file from our server. This action cannot be undone.
